Descripción El sistema deberá comportarse tal como se describe en el siguiente caso de uso cuando Inciar el programa
Precondición PD
Secuencia normal Paso Acción
1 El sistema inica el programa y aparece la primera ventana para logear
2 El actor Usuario-NoLogeado (ACT-0001) puede elegir entre iniciar como administrador en la parte arriba de la derecha con un boton invisible o como cliente
3 El actor Usuario-NoLogeado (ACT-0001) para acceder como usuario debera escribir el numero de la tarjeta en en campo de texto y una vez se valide que la tarjeta existe saldra una nueva ventana para que introduzcas su numero secreto
4 El actor Usuario-NoLogeado (ACT-0001) para acceder como administrador una vez se de al boton invisible de arriba a la derecha le pedira el nombre administrador y la contraseña de este
Postcondición PD
Excepciones Paso Acción
3 Si falla el numero de la tajeta, el sistema sacara por pantalla un error de "El numero de la tarjeta que ha escrito no existe", a continuación este caso de uso continúa
3 Si si fallo 3 veces el numero secreto, el sistema sacara un mensaje por pantalla "Su tarjeta ha sido bloqueada por el exceso de intentos erroneos" la cual tiene que ser desbloqueada por un administrador, a continuación este caso de uso PD
4 Si falla el login, el sistema sacara por pantalla un error de "El usuario y/o contraseña administrador no son correctos", a continuación este caso de uso continúa

Precondición PD
Secuencia normal Paso Acción
1 El actor Usuario-Logeado (ACT-0002) dara al boton de la ventana principal de "Retirar Dinero"

Excepciones Paso Acción
2 Si el dinero indicado es mayor que el de la cuenta
, el sistema sacará un mensaje por pantalla que indicara que dicha cantidad no hay en la cuenta y el tramite no se ha podido realizar, a continuación este caso de uso PD
3 Si da al boton cancelar, el sistema sacará al usuario a la ventana de inicio, a continuación este caso de uso PD
